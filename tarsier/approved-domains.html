Approved source domains

Why this matters

Tarsier does not copy image files itself. When you click Upload to Wikimedia Commons, it opens the Commons upload form pre-filled with a link to the image at its original host (for example inaturalist-open-data.s3.amazonaws.com or bs.plantnet.org). Commons then fetches the file directly from that host — a feature called upload-by-URL (or copy upload).

For security and licensing reasons, Commons only allows upload-by-URL from a curated allow-list of source domains (the $wgCopyUploadsDomains configuration). If an image’s host is not on that list, the upload will fail with a “not in the allowed list of upload domains” error — even when the license is perfectly fine. This is the single most common reason a Tarsier upload does not go through.

Check a source domain

Tarsier cannot read the live Commons configuration, so this only extracts the host for you. Confirm it against the official list: Commons:Upload tools/wgCopyUploadsDomains.

How a domain gets approved

The allow-list is part of Wikimedia’s server configuration, so adding a domain is a community + sysadmin process rather than something you can change yourself:

  1. Confirm the host reliably serves freely-licensed media (CC0, CC-BY, or CC-BY-SA) and is a stable, trusted source.
  2. Start a discussion at the Commons Village pump (proposals) to establish consensus that the domain should be allowed for copy-upload.
  3. With consensus, file a configuration-change request on Phabricator (tagged for the Commons/MediaWiki-Configuration team) asking to add the domain to $wgCopyUploadsDomains in operations/mediawiki-config.
  4. A maintainer merges the change and it is deployed; the domain then works for upload-by-URL.

If a domain is not approved

You can still add the image to Commons — just not via Tarsier’s one-click URL upload. Options:

  • Download the file from the source, verify its license, then upload it manually at Special:Upload.
  • Use the Upload Wizard, which accepts local files.
  • Request that the domain be added (see above) so future uploads work directly.
